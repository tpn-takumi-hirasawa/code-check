# .claude/skills

## 役割

このディレクトリには、複数の手順をまとめた再利用可能な作業フローを定義します。

## スキルとは

スキルは、特定のタスクを効率的に実行するための手順書です。

### スキルの特徴

- **再利用可能**: 類似のタスクで何度も使用できる
- **段階的**: 複数のステップから構成
- **チェックリスト付き**: 実行時の確認項目を含む
- **ベストプラクティス**: チーム内で最適な方法を集約

## 予想されるスキル

以下は、このプロジェクトで定義される可能性のあるスキルです：

| スキル | ファイル | 説明 |
|------|--------|------|
| 新機能作成 | `create-feature/` | 新機能開発の標準フロー |
| バグ修正 | `fix-bug/` | バグ修正の標準フロー |
| PR 準備 | `prepare-pr/` | Pull Request 作成準備 |
| リリース | `release/` | リリース準備と実行 |
| ドキュメント更新 | `update-docs/` | ドキュメント更新フロー |

## スキルの構成

各スキルは、専用のディレクトリを持ちます：

```
.claude/skills/
├── README.md (このファイル)
├── create-feature/
│   ├── README.md
│   ├── checklist.md
│   └── examples/
├── fix-bug/
│   ├── README.md
│   ├── checklist.md
│   └── examples/
└── prepare-pr/
    ├── README.md
    ├── checklist.md
    └── examples/
```

## スキルの使用方法

### CLI での利用

```bash
# スキルの実行を開始
claude /skill [skill-name]
```

### ドキュメントでの参照

ドキュメント内でスキルを参照：

```markdown
新機能開発については、`create-feature` スキルを参照してください。

詳細: `.claude/skills/create-feature/`
```

## スキル管理

### スキル追加時

1. 新しいディレクトリとファイルを作成
2. 詳細な手順を記載
3. チェックリストを提供
4. 実行例を含める
5. README.md に追加

### スキル更新時

1. 実行経験に基づいて改善
2. チェックリスト項目を追加・削除
3. よくある問題を記録
4. 実行例を更新

### スキル廃止時

1. 廃止理由を記録
2. 代替スキルを提示
3. 古いファイルをアーカイブ

## スキル例

### 例 1: 新機能開発スキル

```markdown
# create-feature Skill

## 概要

**スキル名**: `create-feature`

**目的**: 新機能開発を効率的に進める

**対象**: 仕様から実装、テストまでの全工程

## 前提条件

- [ ] 仕様書が承認されている
- [ ] 設計書が承認されている

## スキル実行フロー

### フェーズ 1: 計画

1. 実装計画を作成
2. ファイル構成を確認
3. テスト計画を立案

### フェーズ 2: 実装

1. コードを実装
2. ユニットテストを作成
3. 統合テストを実行

### フェーズ 3: レビュー

1. コードレビューを実施
2. セキュリティレビューを実施
3. ドキュメントを更新

### フェーズ 4: テスト

1. テストが成功することを確認
2. 回帰テストを実施
3. テスト結果を記録

### フェーズ 5: PR

1. PR を作成
2. レビュー コメントに対応
3. PR をマージ
```
