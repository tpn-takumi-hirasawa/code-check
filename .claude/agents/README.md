# .claude/agents

## 役割

このディレクトリには、Claude Code のカスタムエージェントを定義します。

各エージェントは特定の役割を担当し、専門的な観点からプロジェクトに貢献します。

|-----------|------|--------|--------|
| `requirements-analyst` | 要求の整理、前提・制約・未決事項の抽出 | `requirements-analyst.md` | 完了 |
| `spec-writer` | 機能仕様書の作成・更新 | `spec-writer.md` | 完了 |
| `design-writer` | 技術設計書の作成・更新 | `design-writer.md` | 完了 |
| `implementation-planner` | 実装対象と実装順序の整理 | `implementation-planner.md` | 完了 |

## エージェント定義ファイルの構成

各エージェント定義ファイルは、以下の構成を持ちます：

### 1. エージェント概要

- **名前**: エージェントの識別子
- **役割**: エージェントの責務
- **専門分野**: 対象とする分野

### 2. 入出力仕様

- **入力**: エージェントが受け取る情報
- **出力**: エージェントが提供する成果物

### 3. 実行手順

- **前提条件**: エージェント実行前に満たすべき条件
- **実行ステップ**: 具体的な手順
- **検証チェックリスト**: 完了条件の確認項目

### 4. 制約と注意点

- **禁止事項**: してはいけないこと
- **制限事項**: エージェントの制限範囲
- **注意点**: 特に気を付けるべきこと

### 5. 参考資料

- 関連ドキュメントへの参照

## エージェント間の協力

複数のエージェントが同じプロジェクトで協力する場合の原則：

1. **役割の明確化**: 各エージェントの責務を明確に分離
2. **情報の共有**: 前段のエージェント出力が後段の入力となる
3. **品質確保**: 各段階でレビューを実施

### エージェント連携例

```
requirements-analyst
       ↓
   [要求整理結果]
       ↓
  spec-writer
       ↓
  [仕様書]
       ↓
 design-writer
       ↓
 [設計書]
       ↓
implementation-planner
       ↓
  [実装計画]
```

## エージェント定義の更新

エージェント定義は、実際の利用経験に基づいて改善されるべきです。

- 定期的にレビューし、改善点を反映
- チームからのフィードバックを取り入れ
- 実運用で得た知見を共有
