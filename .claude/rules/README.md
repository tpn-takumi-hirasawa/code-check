## ルール定義ファイルの構成

各ルール定義ファイルは、以下の構成を持ちます：

### 1. ルール概要

- **領域**: ルールが適用される領域
- **目的**: ルール制定の目的
- **適用範囲**: どのファイル、どのモジュールに適用されるか

### 2. 基本原則

- **原則 1**: 基本的な方針
- **原則 2**: 基本的な方針

### 3. 具体的ルール

- 詳細なコーディング規約
- 使用すべき/避けるべき構造
- ファイル構成
- 命名規則

### 4. チェックリスト

- 実装時の確認項目

### 5. 参考資料

- 関連ドキュメント、ベストプラクティスガイド

## ルール定義テンプレート

## ルール管理

### ルール追加時

1. 新しいルールファイルを作成
2. 理由と背景を明記
3. 関連する全チーム メンバーでレビュー
4. ドキュメントに追加

### ルール更新時

1. 更新内容と理由を記録
2. 既存コードへの影響を確認
3. 移行期間を設ける（必要に応じて）
4. CLAUDE.md に反映

### ルール廃止時

1. 廃止理由を記録
2. 移行計画を立案
3. 古いファイルをアーカイブ

## ルール適用

Claude Code が提供するコードレビュー、テスト作成、コード改善などの機能利用時に、これらのルールが自動的に参照されます。
