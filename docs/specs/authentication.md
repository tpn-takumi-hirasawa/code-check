# 認証機能 仕様書

## 1. 目的

ユーザーの身元を確認し、適切な権限付与を行うための認証機能を提供する。

## 2. 対象範囲

本仕様書は、Web アプリケーションに対するユーザー認証に関する機能要件を定義します。

### 2.1 対象に含む機能

- ユーザーのログイン・ログアウト
- 認証状態の管理
- セッション管理
- 認証情報の検証

### 2.2 対象に含まない機能

- 認可（アクセス制御）- 別途仕様書で定義予定
- ユーザー登録・アカウント管理 - 別途仕様書で定義予定
- パスワードリセット - 別途仕様書で定義予定
- 多要素認証（MFA） - TBD
- OAuth、SAML など外部認証プロバイダの連携 - TBD

## 4. 前提条件

### 4.1 技術要件

- ユーザー情報はデータベースに格納されている

## 5. 機能要件

### FR-001: ログイン機能

ユーザーが、ユーザー ID とパスワードを入力することで、認証を行い、セッションを開始できる。

**詳細**:
- ユーザー ID とパスワードをシステムに送信
- 送信されたユーザー ID とパスワードが登録情報と一致するか検証
- 一致した場合、セッションを生成し、ユーザーに返却

### FR-002: ログアウト機能

ユーザーがセッションを終了できる。

**詳細**:
- ユーザーがログアウトを要求
- システムが現在のセッションを無効化

### FR-003: 認証状態の確認

システムが、現在のセッションが有効かを確認できる。

**詳細**:
- システムがセッション ID を受け取る
- システムが、そのセッション ID が有効かどうかを判定
- 有効な場合、そのセッションに紐付くユーザー情報を返却

### FR-004: セッション有効期限管理

システムが、有効期限が切れたセッションを管理できる。

**詳細**:
- セッション有効期限が切れたセッションへのアクセスは拒否される
- TBD: セッション有効期限の具体的な値、延長ロジック

## 7. 正常フロー

### 7.1 ログイン成功フロー

```
1. ユーザーがユーザー ID とパスワードを入力
2. システムがユーザー ID をデータベースで検索
3. 検索結果が存在し、パスワードが一致する
4. システムがセッション ID を生成
5. システムがセッション情報をデータベースに保存
6. ユーザーにセッション ID を返却
7. ユーザーがセッション ID を使用して、以後のリクエストを送信
```

### 7.2 認証状態確認フロー

```
1. ユーザーがセッション ID を含むリクエストを送信
2. システムがセッション ID をデータベースで検索
3. セッション ID が有効かつ有効期限内である
4. システムがセッション情報とユーザー情報を返却
5. ユーザーの認証が確認される
```

### 7.3 ログアウトフロー

```
1. ユーザーがログアウトを要求
2. システムが現在のセッション ID を受け取る
3. システムがデータベースのセッション情報を削除または無効化
4. ユーザーに削除が完了したことを通知
```

## 8. 例外フロー

### 8.1 ユーザー ID が見つからない

```
1. ユーザーが存在しないユーザー ID を入力
2. システムがユーザー ID をデータベースで検索
3. 検索結果が存在しない
4. システムがエラーを返却（詳細は「10. エラー処理」を参照）
```

### 8.2 パスワードが不正

```
1. ユーザーがユーザー ID とパスワードを入力
2. システムがユーザー ID をデータベースで検索
3. ユーザー ID は存在するがパスワードが一致しない
4. システムがエラーを返却
```

### 8.3 セッション有効期限切れ

```
1. ユーザーがセッション ID を含むリクエストを送信
2. システムがセッション ID をデータベースで検索
3. セッション ID は存在するが有効期限が切れている
4. システムがセッション無効エラーを返却
```

### 8.4 セッション ID が見つからない

```
1. ユーザーが存在しないセッション ID を含むリクエストを送信
2. システムがセッション ID をデータベースで検索
3. 検索結果が存在しない
4. システムがセッション無効エラーを返却
```

### 8.5 セッション無効化後のアクセス

```
1. ユーザーがログアウト後、同じセッション ID でリクエスト送信
2. システムがセッション ID をデータベースで検索
3. セッション情報が削除されている
4. システムがセッション無効エラーを返却
```

## 10. 権限

認証機能に関する権限は以下の通りです。

| 操作 | 誰が実行可能か |
|-----|-------------|
| ログイン | 誰でも |
| ログアウト | 認証済みユーザー |
| 認証状態確認 | 誰でも |
| セッション情報削除 | システム管理者、またはシステム |

**注記**: 認可（アクセス制御）の詳細は別途仕様書で定義します。

## 11. エラー処理

### 11.1 ログイン関連のエラー

| エラーコード | エラーメッセージ | HTTP ステータス | 対応 |
|-----------|-----------------|-----------------|------|
| AUTH_001 | ユーザー ID またはパスワードが不正です | 401 | 入力をやり直すよう指示 |
| AUTH_002 | ユーザー ID が見つかりません | 401 | 入力をやり直すよう指示 |
| AUTH_003 | パスワードが不正です | 401 | 入力をやり直すよう指示 |
| AUTH_004 | ログイン試行回数が超過しました | 429 | 一定時間後に再度試すよう指示（TBD） |

### 11.2 セッション関連のエラー

| エラーコード | エラーメッセージ | HTTP ステータス | 対応 |
|-----------|-----------------|-----------------|------|
| SESSION_001 | セッション ID が見つかりません | 401 | 再度ログインするよう指示 |
| SESSION_002 | セッションの有効期限が切れています | 401 | 再度ログインするよう指示 |
| SESSION_003 | セッション情報を取得できません | 500 | システム管理者に報告 |

### 11.3 一般的なエラー

| エラーコード | エラーメッセージ | HTTP ステータス | 対応 |
|-----------|-----------------|-----------------|------|
| SYSTEM_001 | システムエラーが発生しました | 500 | ログを記録し、管理者に通知 |

## 13. 受入条件

### AC-001: ログイン成功

正しいユーザー ID とパスワードでログインすると、セッション ID が返却される。

**テスト手順**:
1. 正しいユーザー ID とパスワードを入力
2. ログインボタンをクリック
3. セッション ID が返却される

**期待結果**: セッション ID が返却され、ユーザーが認証状態になる

### AC-002: ログイン失敗（ユーザー ID 不正）

存在しないユーザー ID でログインすると、エラーメッセージが表示される。

**テスト手順**:
1. 存在しないユーザー ID を入力
2. 正しいパスワードを入力
3. ログインボタンをクリック

**期待結果**: エラーメッセージ「ユーザー ID またはパスワードが不正です」が表示される

### AC-003: ログイン失敗（パスワード不正）

正しいユーザー ID で間違ったパスワードを入力してログインすると、エラーメッセージが表示される。

**テスト手順**:
1. 正しいユーザー ID を入力
2. 間違ったパスワードを入力
3. ログインボタンをクリック

**期待結果**: エラーメッセージ「ユーザー ID またはパスワードが不正です」が表示される

### AC-004: ログアウト成功

ログアウトボタンをクリックすると、セッションが終了される。

**テスト手順**:
1. ユーザーがログイン済み状態
2. ログアウトボタンをクリック
3. セッション ID を含むリクエストを送信

**期待結果**: セッション無効エラーが返却される

### AC-005: 認証状態確認（有効なセッション）

有効なセッション ID でリクエストを送信すると、認証状態が確認される。

**テスト手順**:
1. ログイン後、有効なセッション ID を取得
2. セッション ID を含むリクエストを送信
3. システムが認証状態を確認

**期待結果**: ユーザー情報が返却され、認証が確認される

### AC-006: 認証状態確認（無効なセッション）

無効なセッション ID でリクエストを送信すると、エラーが返却される。

**テスト手順**:
1. 存在しないセッション ID を含むリクエストを送信
2. システムが認証状態を確認

**期待結果**: セッション無効エラーが返却される

### AC-007: セッション有効期限切れ

有効期限が切れたセッション ID でリクエストを送信すると、エラーが返却される。

**テスト手順**:
1. ログイン後、セッション ID を取得
2. セッション有効期限が切れるまで待機（TBD）
3. セッション ID を含むリクエストを送信

**期待結果**: セッション有効期限切れエラーが返却される

### AC-008: 入力検証（ユーザー ID 未入力）

ユーザー ID が未入力でログインボタンをクリックすると、バリデーションエラーが表示される。

**テスト手順**:
1. ユーザー ID を入力せず、パスワードのみ入力
2. ログインボタンをクリック

**期待結果**: バリデーションエラーメッセージ「ユーザー ID が入力されていません」が表示される

### AC-009: 入力検証（パスワード未入力）

パスワードが未入力でログインボタンをクリックすると、バリデーションエラーが表示される。

**テスト手順**:
1. ユーザー ID を入力し、パスワードを入力しない
2. ログインボタンをクリック

**期待結果**: バリデーションエラーメッセージ「パスワードが入力されていません」が表示される
